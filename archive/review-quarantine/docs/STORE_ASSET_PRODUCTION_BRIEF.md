# Store Asset Production Brief
## Civic Citizenship

**Purpose:** Execution-ready brief for App Store and Google Play screenshot capture  
**Phase:** Phase 4 Day 3  
**Target Capture Day:** March 30, 2026  
**Launch Window:** April 1-7, 2026

---

## Asset Scope

### Required Deliverables
- iOS screenshots: 5 core images at 1170x2532
- Android screenshots: 5 core images at 1080x1920
- Android feature graphic: 1024x500 PNG
- Optional preview video: 15-30 seconds

### Creative Direction
- Clean, trustworthy, academic
- Emphasize clarity, progress, confidence
- Show real app UI, not mock fantasy screens
- Use short text overlays only
- Keep one core message per asset

### Visual Rules
- Use live app screens from the current build
- Avoid tiny text or cluttered overlays
- Keep status bar clean where possible
- Use consistent overlay typography and spacing
- Prefer bright, high-contrast screens that show function immediately

## Capture Setup

### Before Capturing
- Use seeded test details with realistic date and location
- Ensure no debug UI is visible
- Use stable sample data for stats/progress
- Use bright, consistent device theme
- Disable distracting notifications
- Make sure banner ads do not obscure the key UI in screenshots

### Recommended Test Data State
- User name: Future Citizen
- Test type: Naturalization (128Q)
- Test date: 21-30 days in future
- Progress stats visible
- Interview CTA visible on Home
- At least one completed quiz session in history
- At least one completed interview session in history

## Shot List

### Shot 1: Home Screen / Entry Point
**Goal:** Show the app is organized, motivating, and test-focused.

**Screen:** HomeScreen  
**Device:** iOS + Android  
**Capture State:**
- Profile header visible
- Test details visible
- Main quiz CTA visible
- Interview CTA visible
- Progress/study card visible if possible

**Overlay Copy:**
- Headline: Ace your civics test
- Subhead: Practice quizzes and interview prep in one app

**Why This Shot Matters:**
- Establishes the app's value immediately
- Shows both major learning modes from one screen

**Capture Notes:**
- Keep Interview CTA fully visible
- Avoid clipping bottom cards awkwardly
- Best if header and CTA buttons are both in frame

---

### Shot 2: Quiz Question Interface
**Goal:** Show the core quiz experience clearly.

**Screen:** QuizScreen  
**Device:** iOS + Android  
**Capture State:**
- One multiple-choice question on screen
- Answer options visible
- Progress bar visible
- Image/question card visible if available

**Overlay Copy:**
- Headline: Practice real civics questions
- Subhead: Quick multiple-choice drills with instant feedback

**Why This Shot Matters:**
- Confirms the app has strong core study functionality
- Helps store users understand the primary daily workflow

**Capture Notes:**
- Use a visually clear question with readable answer options
- Prefer a question with image support if it looks cleaner
- Avoid states where answer is already selected unless it looks intentional

---

### Shot 3: Quiz Feedback / Results
**Goal:** Show learning reinforcement and correction.

**Screen:** QuizScreen feedback state or ReviewScreen  
**Device:** iOS + Android  
**Capture State:**
- Correct/incorrect feedback visible
- Explanation visible if readable
- Positive reinforcement visible

**Overlay Copy:**
- Headline: Get instant feedback
- Subhead: Learn what is right and why it matters

**Why This Shot Matters:**
- Demonstrates educational value beyond simple flashcards
- Makes the app look helpful, not just test-like

**Capture Notes:**
- Prefer a screen where the explanation is concise and readable
- Keep emotional tone encouraging, not punitive

---

### Shot 4: Interview Mode Recording
**Goal:** Highlight the unique feature.

**Screen:** InterviewScreen recording or prompt phase  
**Device:** iOS + Android  
**Capture State:**
- Interview title visible
- Current question visible
- Recording interface visible
- Clean, focused layout

**Overlay Copy:**
- Headline: Train in Interview Mode
- Subhead: Answer with your voice and practice with confidence

**Why This Shot Matters:**
- This is the differentiator versus generic civics apps
- Should be one of the strongest screenshots in the set

**Capture Notes:**
- Prefer prompt or ready-to-record state over loading state
- If recording UI is animated, capture a stable still frame

---

### Shot 5: Interview Score / Follow-Up Support
**Goal:** Show adaptive coaching.

**Screen:** Interview scoring phase or follow-up phase  
**Device:** iOS + Android  
**Capture State:**
- Grade or score visible
- Feedback visible
- Optional follow-up question visible if it looks clearer

**Overlay Copy:**
- Headline: Improve with follow-up questions
- Subhead: Get coaching when an answer needs work

**Why This Shot Matters:**
- Shows the app is adaptive, not static
- Reinforces the educational coaching loop

**Capture Notes:**
- If the score screen looks stronger than follow-up, use score screen
- If follow-up screen is more distinctive, use that instead

---

### Shot 6: Progress / Profile
**Goal:** Show retention and habit-building.

**Screen:** ProfileScreen or progress-heavy Home state  
**Device:** iOS + Android  
**Capture State:**
- Accuracy, streak, or progress cards visible
- Clear stats, not empty states

**Overlay Copy:**
- Headline: Stay on track every day
- Subhead: Track accuracy, streaks, and study progress

**Why This Shot Matters:**
- Adds proof of long-term value
- Helps communicate daily practice habit

**Capture Notes:**
- Use only if progress UI is visually strong
- If ProfileScreen is weak, use Home with stats visible instead
